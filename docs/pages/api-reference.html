Vehicles

GET /vehicles

Retrieve a list of all vehicles with their current status and position.

Query Parameters

Parameter Type Description
status string Filter by status (active, idle, maintenance)
limit integer Maximum number of results (default: 50)

Response Example

{
  "vehicles": [
    {
      "id": "VH-1001",
      "lat": 25.2048,
      "lng": 55.2708,
      "status": "active",
      "speed": 45.5,
      "heading": 180,
      "driver": "Ahmed Hassan",
      "current_route_id": "RT-5432"
    }
  ],
  "total": 156
}
GET /vehicles/{id}

Get detailed information about a specific vehicle.

Path Parameters

Parameter Type Description
id * string Unique vehicle identifier
GET /vehicles/{id}/history

Get the location history for a vehicle over a time period.

Route Optimization

POST /optimization/start

Start a new route optimization job with specified constraints.

Request Body

{
  "vehicles": ["VH-1001", "VH-1002"],
  "deliveries": [
    {
      "id": "DEL-001",
      "lat": 25.1972,
      "lng": 55.2744,
      "time_window": ["09:00", "12:00"],
      "demand": 10
    }
  ],
  "strategy": "PATH_CHEAPEST_ARC",
  "max_iterations": 10000
}
GET /optimization/{run_id}

Get the status and results of an optimization job.

Risk Assessment

GET /risk/scores

Get current risk scores for all vehicles.

Response Example

{
  "scores": [
    {
      "vehicle_id": "VH-1001",
      "overall_score": 0.35,
      "level": "low",
      "factors": {
        "weather": 0.2,
        "traffic": 0.4,
        "driver_fatigue": 0.3,
        "vehicle_health": 0.1
      }
    }
  ]
}
GET /risk/alerts

Get all active risk alerts.

WebSocket Events

Connect to ws://localhost:8000/ws for real-time updates.

WS vehicle:update Server → Client

Real-time vehicle position updates sent every second.

{
  "id": "VH-1001",
  "lat": 25.2048,
  "lng": 55.2708,
  "status": "active",
  "speed": 45.5,
  "timestamp": "2024-01-21T10:30:00Z"
}

WS optimization:progress Server → Client

Optimization algorithm iteration updates.

{
  "run_id": "opt-12345",
  "iteration": 5420,
  "best_cost": 12450.5,
  "progress": 0.54
}

WS risk:alert Server → Client

Risk threshold exceeded alert.

{
  "vehicle_id": "VH-1003",
  "level": "high",
  "factors": ["driver_fatigue", "weather"],
  "score": 0.82
}

WS message:new Bidirectional

New chat message between driver and dispatcher.

{
  "from": "dispatcher-001",
  "to": "driver-VH1001",
  "content": "Update on delivery DEL-5432",
  "timestamp": "2024-01-21T10:35:00Z"
}
