Quick Start Guide

Getting Started

Get the Logistics AI Platform running locally in minutes with Docker. Follow this step-by-step guide to set up your development environment.

Step 0

Prerequisites

Make sure you have these tools installed before proceeding

Docker Desktop

Version 20.10 or higher

Docker Compose

Version 2.0 or higher

Git

Version 2.30 or higher

System Resources

8GB RAM minimum, 16GB recommended

Required API Keys

You'll need to obtain these API keys before starting:

Mapbox Token

For interactive maps

Get free token →

Gemini API Key

For AI features

Get free key →

1 Clone the Repository

Start by cloning the project repository to your local machine:

Terminal
# Clone the repository git clone https://github.com/Div-9898/AIO_Final_Project.git # Navigate to project directory cd AIO_Final_Project

2 Configure Environment Variables

Create a .env file in the project root with your API keys:

.env
# Required API Keys MAPBOX_TOKEN=your_mapbox_token_here GEMINI_API_KEY=your_gemini_api_key_here # Optional: Override default database credentials # POSTGRES_USER=postgres # POSTGRES_PASSWORD=postgres # POSTGRES_DB=logistics # NEO4J_AUTH=neo4j/password

Environment Variables Reference

Variable Required Description
MAPBOX_TOKEN Required Mapbox GL access token for interactive maps
GEMINI_API_KEY Required Google Gemini API key for AI features
DATABASE_URL Auto PostgreSQL connection string (auto-configured by Docker)
REDIS_URL Auto Redis connection string (auto-configured by Docker)
NEO4J_URL Auto Neo4j bolt URL (default: bolt://neo4j:7687)

3 Start the Platform

Use the management script to start all services:

Terminal
# Make start script executable (first time only) chmod +x start.sh # Start all services ./start.sh start

The platform will automatically:

  • Pull and build all Docker images
  • Initialize PostgreSQL, Redis, and Neo4j databases
  • Start the FastAPI backend with hot reload
  • Launch the Next.js frontend
  • Run the vehicle simulation engine

4 Access the Application

Once all services are running, open your browser to access:

Dashboard

localhost:3000

API Docs

localhost:8000/docs

Neo4j Browser

localhost:7474

Health Check

localhost:8000/health
Reference

Management Commands

All available commands for managing the platform

Service Management

./start.sh start Start all services
./start.sh stop Stop all services
./start.sh restart Restart all services
./start.sh status Check service health

Development

./start.sh dev Start with live reload
./start.sh logs Follow all logs
./start.sh logs backend Follow specific service logs
./start.sh build Rebuild all images

Database Access

./start.sh db Open PostgreSQL CLI
./start.sh redis Open Redis CLI
./start.sh clean Remove all containers & volumes
Help

Troubleshooting

Common issues and their solutions

Port Already in Use

If you see an error about ports being in use, find and kill the process:

lsof -i :3000 kill -9 <PID>

Docker Memory Issues

Increase Docker memory allocation in Docker Desktop settings to at least 8GB for smooth operation.

Database Connection Errors

Reset all databases and try again:

./start.sh clean ./start.sh start

Mapbox Not Loading

Verify your MAPBOX_TOKEN is set correctly in .env and has the required scopes (styles:read, fonts:read).

WebSocket Connection Failed

Ensure the backend is fully started by checking the health endpoint at http://localhost:8000/health.
